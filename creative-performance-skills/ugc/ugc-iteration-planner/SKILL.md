# UGC Iteration Planner

**Category:** ugc
**Tier:** pro

## Purpose

This Skill builds a structured iteration plan for UGC creative — a systematic approach to taking a single winning UGC concept or creator and generating maximum testing value from minimal production cost. UGC iteration is the highest-leverage activity in a performance creative program: one creator, one shoot, multiple angles, hooks, and formats produced from a single session. This Skill plans that session for maximum output.

---

## When to Use

- You have a winning UGC ad and want to iterate on it to find the performance ceiling
- You're scheduling a creator shoot and want to plan the maximum number of testable variations
- You want to test multiple hook options on the same body copy and visual
- You're building a UGC test bank across multiple angles from a single creator
- You want to plan a systematic UGC testing program for the next 30-60 days
- A UGC creator has limited availability and you want to maximize the session's output

---

## Inputs Required

- Winning or base UGC concept (description or link to the existing ad)
- What is currently working (which element is confirmed performing: hook rate, CVR, etc.)
- What has NOT been tested yet (angles, formats, hooks not yet tried)
- Creator availability and shoot constraints (single session or multiple?)
- Target platform and formats (Reels, Feed, Stories — aspect ratios matter)
- Budget for this creator engagement
- Number of distinct testable variations desired
- Optional: existing test history showing what hook angles or formats have been tested

---

## What This Skill Does

Builds a complete UGC iteration plan: a structured shot list and brief expansion for a single creator session that produces the maximum number of properly isolated, testable creative variations.

---

## Analysis Workflow

1. **Audit the base creative**
   Identify what elements are confirmed and should be protected in iterations: hook? body copy structure? proof point? CTA? Visual direction? Only iterate on unconfirmed variables.

2. **Apply the UGC iteration hierarchy**
   Prioritize iterations in this order (most bang per production effort):
   - Layer 1: Hook variations (same body/CTA, different opening 3-5 seconds) — highest testing value, lowest production cost
   - Layer 2: Angle variations (same format, different emotional angle) — high value, moderate effort
   - Layer 3: Proof variations (same angle, different proof element) — moderate value, low effort
   - Layer 4: CTA variations (same creative, different CTA) — low-effort testing lever
   - Layer 5: Format variations (same content, reframed for different placement) — production-level change

3. **Plan hook variation set**
   Write 3-5 distinct hook scripts for the same base concept. Each hook must:
   - Be under 5 seconds
   - Use a different entry point (question, statement, action, number-led, quote-drop)
   - Target the same awareness stage
   - Be filmable with no additional production (same location, same wardrobe, just different opening line and action)

4. **Plan angle variation set**
   Write 2-3 angle variations that change the emotional entry point but keep the same offer, proof, and CTA:
   - Angle A: Problem-first
   - Angle B: Outcome-first
   - Angle C: Skepticism/objection-first

5. **Plan the shoot breakdown**
   Map every required shot to a timestamp and scene number. Organize so the creator can shoot all hooks consecutively, then all main content, minimizing setup changes. Output as a shot list.

6. **Plan the edit structure**
   Document exactly how the filmed pieces will be assembled into distinct creative units. Variation A = [Hook Option 1] + [Body Scene 1-4] + [CTA]. Variation B = [Hook Option 2] + [Body Scene 1-4] + [CTA]. This is the modular editing plan.

7. **Calculate expected output**
   Count total distinct testable variations: (hooks × angle variations × formats). Confirm the iteration plan delivers the desired number of variations.

8. **Write the session brief**
   Consolidate everything into a single shoot-day brief the creator can follow: shot order, hook scripts, scene descriptions, total duration target, and any on-set direction notes.

---

## Output Requirements

**Iteration Map**
Table: Variation # | Hook Used | Angle | Proof | CTA | Format | Test Variable

**Hook Script Set**
3-5 hook scripts, each with delivery note and estimated hook duration.

**Shoot Day Shot List**
Organized by scene type (all hooks first, then body, then CTAs), with estimated duration and any location/framing notes.

**Modular Edit Plan**
How each variation is assembled from the filmed components.

**Expected Output Count**
Total number of distinct testable variations from this one session.

---

## Best Practices Applied

- Hook-first iteration order maximizes testing value for minimum production cost
- Modular filming (hooks shot separately from body content) enables combinatorial variation without reshooting
- Protecting confirmed elements prevents removing what's already working from iterations
- The edit plan is documented before the shoot — editing decisions are not made on a whim
- One session should produce enough variations for a 4-6 week testing cycle

---

## Guardrails

- Do not plan iterations that change multiple variables simultaneously in the same variation
- Do not iterate on the protected variable — if the hook is what's winning, don't test a new hook format until body copy has been optimized
- Do not plan a shoot that produces more variations than the budget can test meaningfully
- Do not forget to include a "control" version in the iteration plan — the original must remain in rotation during testing
- Do not overlook format-specific considerations — a 9:16 Reels version requires different framing than a 1:1 feed version
